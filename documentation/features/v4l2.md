# V4L2 Async Capture

V4L2 capture support is available in both models.

## Readiness Model

- epoll-driven readiness wait
- mmap buffer setup and queue management
- `frame_view` auto-requeues buffers on destruction

## Completion Model

- one-shot `IORING_OP_POLL_ADD` wait for fd readiness
- synchronous `VIDIOC_DQBUF` after wake-up (hybrid design)
- explicit poll re-arm per frame fetch

## Device Targets

- USB webcams
- MIPI CSI-2 pipelines
- GMSL camera chains

Optional host tooling (Ubuntu/Debian):

```bash
bash scripts/install_v4l2_deps.sh
```

Or via umbrella bootstrap:

```bash
bash scripts/bootstrap_optional_deps.sh --v4l2
```

`capture::create()` performs the full V4L2 setup sequence (`QUERYCAP`, `S_FMT`, `S_PARM`, `REQBUFS`, `QUERYBUF`/`mmap`, enqueue, `STREAMON`) and returns a streaming-ready capture object.

Completion mode remains intentionally hybrid by kernel API design: io_uring handles readiness notification, then dequeue happens through synchronous V4L2 ioctl.

## C++ Key Methods - Readiness model

```cpp
#include <cstddef>
#include <kmx/aio/readiness/executor.hpp>
#include <kmx/aio/readiness/v4l2/capture.hpp>
#include <span>

using namespace kmx::aio;

task<void> capture_frames(readiness::executor& exec)
{
	auto cap = readiness::v4l2::capture::create(exec,
												{
													.device       = "/dev/video0",
													.format       = readiness::v4l2::fourcc::nv12,
													.size         = {1920u, 1080u},
													.fps          = {1u, 30u},
													.buffer_count = 4u,
												});
	if (!cap)
		co_return;

	// cap->config() reflects the negotiated format/resolution/fps/buffer_count after setup.

	while (true)
	{
		auto frame = co_await cap->next_frame();
		if (!frame)
			break;

		std::span<const std::byte> pixels = frame->data();
		const auto& meta = frame->metadata();

		process_frame(pixels, meta.width, meta.height, meta.timestamp_ns);
	}
}


// Typical readiness entry point
readiness::executor_config cfg{ .thread_count = 1u };
readiness::executor exec(cfg);
exec.spawn(capture_frames(exec));
exec.run();
```

## C++ Key Methods - Completion model

```cpp
#include <kmx/aio/completion/executor.hpp>
#include <kmx/aio/completion/v4l2/capture.hpp>
#include <memory>

using namespace kmx::aio;

task<void> capture_frames(std::shared_ptr<completion::executor> exec)
{
	auto cap = completion::v4l2::capture::create(exec,
												 {
													 .device       = "/dev/video0",
													 .format       = completion::v4l2::fourcc::yuyv,
													 .size         = {1280u, 720u},
													 .fps          = {1u, 30u},
													 .buffer_count = 4u,
												 });
	if (!cap)
		co_return;

	while (true)
	{
		auto frame = co_await cap->next_frame();
		if (!frame)
			break;

		process_frame(
			frame->data(), frame->metadata().width, frame->metadata().height, frame->metadata().timestamp_ns
		);
	}
}

// Typical completion entry point
auto exec = std::make_shared<completion::executor>();
exec->spawn(capture_frames(exec));
exec->run();
```

## Example

Run readiness and completion capture samples:

```bash
READINESS_BIN="$(find debug -type f -name sample-v4l2-capture | head -n 1)"
COMPLETION_BIN="$(find debug -type f -name sample-v4l2-completion-capture | head -n 1)"
"$READINESS_BIN" --max-frames 10
"$COMPLETION_BIN" --max-frames 10
```
